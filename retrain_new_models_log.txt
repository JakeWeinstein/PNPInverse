  2. GP SURROGATE TRAINING
===========================================================================

Command:
  python scripts/surrogate/train_gp.py \
    --training-data data/surrogate_models/training_data_merged.npz \
    --split-indices data/surrogate_models/split_indices.npz \
    --output-dir data/surrogate_models/gp

Configuration:
  - 44 independent exact GPs (22 CD + 22 PC), Matern 5/2 ARD kernel
  - 2967 train / 524 test samples, 22 voltage points
  - Max Adam iterations: 200, LR: 0.1, early stopping patience: 20
  - Parallelized with joblib (10 workers)

Training time: 499.3s (~8.3 min)

Validation metrics (test set, 524 samples):
  Current Density:
    RMSE:               1.950e-02
    Max absolute error: 7.490e-01
    Mean NRMSE:         3.85%
  Peroxide Current:
    RMSE:               2.655e-02
    Max absolute error: 7.665e-01
    Mean NRMSE:         1610.96%  (inflated by near-zero PC samples)

UQ Calibration (predictive intervals):
     Level    CD coverage    PC coverage
      50%        74.2%          59.7%
      80%        84.9%          79.4%
      90%        89.2%          82.2%
      95%        90.1%          84.3%
  CD 90% interval: WELL-CALIBRATED (89.2% coverage)
  PC 90% interval: OVERCONFIDENT (82.2% coverage, below 85%)

Save/Load Round-Trip: PASS (max abs diff = 0.0 for both CD and PC)

Gradient Accuracy (autograd vs central finite differences):
  Result: 0/10 points passed (<1% relative error)
  All 10 test points showed ~100% relative error between autograd
  and FD gradients. This is caused by GPyTorch's CG-based approximate
  inference during prediction (NumericalWarnings about CG convergence
  were abundant). The fast_pred_var setting uses conjugate gradient
  approximations that break the autograd gradient chain. This is a
  known limitation for exact GP models with large training sets (N=2967).

Model saved to: data/surrogate_models/gp/

  3. PCE SURROGATE TRAINING
===========================================================================

Command:
  python scripts/surrogate/train_pce.py \
    --training-data data/surrogate_models/training_data_merged.npz \
    --split-indices data/surrogate_models/split_indices.npz \
    --output-dir data/surrogate_models/pce \
    --max-degree 5 --skip-convergence

Note: Initial run with default max-degree 6 and convergence study was
killed after 25+ minutes (LOO cross-validation with hat matrix was
extremely slow for large N=2967 with high-degree polynomial bases).
Re-ran with --max-degree 5 and --skip-convergence to reduce wall time.

===========================================================================
  5. BUGS FIXED
===========================================================================

None -- all scripts ran without code bugs. The only issue was performance:
the PCE LOO cross-validation was too slow at default settings (degree
candidates 3-8 on 2967 samples). Resolved by passing --max-degree 5
and --skip-convergence. The LOO CV still evaluated all 6 candidate
degrees (3-8) during degree selection, but the convergence study
(which re-fits separate models at each degree and validates on test set)
was skipped to save time.
