**Figure 1: MMS Convergence (mms_convergence.pdf)**
- Two-panel (1x2) log-log plot: L2 errors (left), H1 errors (right)
- 5 fields (O2, H2O2, H+, ClO4-, phi) with distinct colors and markers
- Each line labeled with convergence rate: e.g., "O2 (rate=2.01)"
- Reference slope lines: O(h^2) dashed on L2 panel, O(h) dashed on H1 panel
- X-axis: "Mesh spacing $h$", Y-axis: "$L^2$ error" / "$H^1$ error"
- Grid enabled (alpha=0.3)
- Data source: convergence_data.json -> fields -> {L2_errors, H1_errors} vs metadata.h_values
- figsize=(12, 5) for the two-panel layout

**Figure 2: Parameter Recovery vs Noise (parameter_recovery.pdf)**
- Single-panel bar or marker plot
- X-axis: noise level (0%, 1%, 2%, 5%)
- Y-axis: median max relative error
- Horizontal dashed lines at each gate threshold (0.15, 0.25, 0.30, 0.50)
- Mark 5% noise distinctly (different marker style or hatching) with annotation "informational"
- Note the ~11% surrogate bias floor at 0% noise (horizontal dotted line or annotation)
- Data source: parameter_recovery_summary.json -> results -> {median_max_relative_error, gate_threshold}

**Figure 3: Worst-Case I-V Overlay (worst_case_iv.pdf)**
- Single-panel line plot
- X-axis: applied potential (phi_applied)
- Y-axis: current density
- Plot target_primary vs simulated_primary (and target_secondary vs simulated_secondary)
- Legend distinguishing "PDE target" vs "Surrogate fit" for each observable
- Data source: master_inference_v13/P2_pde_full_cathodic/multi_obs_fit.csv

**Table 1: MMS Convergence Rates (tables/mms_rates.tex)**
- Columns: Species | L2 Rate | L2 R^2 | H1 Rate | H1 R^2 | GCI (finest)
- 5 rows (O2, H2O2, H+, ClO4-, phi)
- Use booktabs formatting (\toprule, \midrule, \bottomrule)
- GCI at finest mesh = last entry in each field's gci array -> gci value
- Format: rates to 3 decimal places, R^2 to 7 decimal places, GCI to 3 decimal places

**Table 2: Surrogate Fidelity (tables/surrogate_fidelity.tex)**
- Columns: Model | CD Median | CD 95th | CD Max | PC Median | PC 95th
- 4 rows (nn_ensemble, rbf_baseline, pod_rbf_log, pod_rbf_nolog)
- Use human-readable model names (NN Ensemble, RBF Baseline, POD-RBF (log), POD-RBF (no-log))
- Report median and 95th percentile as primary (not mean/max for PC due to outliers per Phase 3 decision)
- Include CD max for completeness
- Add table footnote about PC near-zero-range NRMSE inflation
- All NRMSE values formatted to 4 decimal places (or scientific notation if very small)

**Table 3: Parameter Recovery (tables/parameter_recovery.tex)**
- Columns: Noise Level | Median Max Error | Gate Threshold | Status
- 4 rows (0%, 1%, 2%, 5%)
- 5% row in italics with footnote "informational -- exceeds surrogate approximation limits"
- Note surrogate bias (~10.7%) at 0% noise
- Status: Pass/Fail from JSON pass field

**Table 4: Gradient Consistency (tables/gradient_consistency.tex)**
- Two sub-sections: Surrogate FD convergence + PDE FD convergence
- Surrogate section: param | FD convergence rate (from gradient_fd_convergence.json convergence_rates)
- PDE section: evaluation point | param | convergence rate (from gradient_pde_consistency.json, +10% point which has clean rates)
- Note: analytic_vs_fd relative diff is 0.0 for all params (exact match at 1e-3 step)

**Table 5: Summary (tables/summary.tex)**
- Columns: Layer | Test | Key Result | Status
- Rows covering all verification layers:
  - Forward Solver | MMS Convergence | L2~O(h^2), H1~O(h), R^2>0.999 | Pass
  - Forward Solver | GCI Uncertainty | Safety factor ~1.25 at finest mesh | Pass
  - Surrogate | Hold-out Validation | CD median NRMSE < 0.005 (NN Ensemble) | Pass
  - Inverse | Parameter Recovery (0-2% noise) | Median max error < gate | Pass
  - Inverse | Gradient Consistency | FD convergence O(h^2) at +10% | Pass
  - Inverse | Multistart Basin | CV < 0.002, NRMSE < 0.004 | Pass
  - Pipeline | Reproducibility | Bitwise-identical outputs | Pass
