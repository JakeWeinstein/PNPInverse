  <action>
This is the core FWD-02 work. The MMS script currently builds its own inline weak form in each `run_mms_*` function. Refactor it to use production `build_forms()` from `Forward.bv_solver.forms`.

**Strategy: Option A (Thin wrapper for MMS) per RESEARCH.md recommendation.** This minimizes production code changes per PROJECT.md constraint.

**Step 1: Understand the current MMS inline weak form.**
Read each `run_mms_*` function thoroughly. Identify every UFL term: diffusion, electromigration, Poisson/eps_hat, BV boundary flux, source terms. Note the exact coefficient names and forms.

**Step 2: Understand production build_forms().**
Read `Forward/bv_solver/forms.py:build_forms()` thoroughly. Map every term to its MMS counterpart. Identify:
- Time-stepping terms (MMS is steady-state, production has `(c - c_old)/dt * v * dx`)
- Optional features that must be disabled: `regularize_concentration`, `clip_exponent`, `a_vals=[0]*n`, steric terms
- Marker conventions: MMS uses RectangleMesh (1=left, 2=right, 3=bottom, 4=top); production uses configurable markers

**Step 3: Build minimal solver_params for MMS.**
Construct the 11-element solver_params tuple that `build_forms()` expects, configured for MMS:
- `nondim.enabled=True` with `*_inputs_are_dimensionless=True` so MMS nondim params pass through unchanged
- `potential_scale_v=V_T` so `electromigration_prefactor=1.0`
- All optional features disabled: `regularize_concentration=False`, `clip_exponent=False`, `a_vals=[0]*n`
- Correct marker mapping for RectangleMesh: `electrode_marker=3`, `concentration_marker=4`, `ground_marker=4` (or whatever matches the MMS mesh conventions -- verify by reading the mesh setup code)

**Step 4: Refactor run_mms_single_species().**
Replace inline weak form assembly with:
1. Build solver_params with MMS-appropriate config
2. Call `build_context(solver_params, mesh=mms_mesh)` to get ctx
3. Call `build_forms(ctx, solver_params)` to get F_res
4. Handle steady-state: either set dt very large so time-stepping term is negligible, or subtract the time-stepping term from F_res after build_forms returns. The RESEARCH.md recommends the large-dt approach.
5. Add MMS source terms: `F_res -= S_c * v * dx` for each species, `F_res -= S_phi * w * dx`
6. Add MMS boundary corrections as needed

**CRITICAL: Run the EXISTING MMS script first (before refactor) on meshes [8, 16, 32] and record the convergence rates and L2 errors. After refactor, compare to confirm results are identical (or very close). If Newton iteration count changes significantly, investigate per RESEARCH.md pitfall #2.**

**Step 5: Refactor run_mms_two_species() and run_mms_charged().**
Apply the same pattern. Each should use build_forms() instead of inline assembly.

**Step 6: Write the audit document.**
Create `scripts/verification/WEAK_FORM_AUDIT.md` containing:
- Term-by-term correspondence table: MMS term | Production term | Match?
- For each PDE term (diffusion, electromigration, Poisson, BV boundary): show the UFL expression from production build_forms() and confirm MMS now uses it
- Document any differences and why they exist (e.g., MMS adds source terms that production doesn't have)
- Document which optional features were disabled and why
- Document marker convention mapping
- This document feeds into Phase 6 V&V report

**Important constraints:**
- Do NOT modify `Forward/bv_solver/forms.py` unless a bug is discovered (per PROJECT.md). If a bug IS found, fix it and document in the audit.
- Keep the `sys.path` handling as-is if needed for script invocation, but also ensure the module can be imported by pytest (add `__init__.py` to `scripts/verification/` if needed, or use sys.path in the test file).
- Preserve the existing `compute_rates`, `format_table_*`, `plot_convergence`, and `main` functions -- only refactor the `run_mms_*` functions.
