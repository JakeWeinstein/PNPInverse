Chi-squared identifiability (from RESEARCH.md):
- DELTA_CHI2_95 = 3.84 (chi2(1) 95th percentile, 1 DOF)
- chi2_profile = (J_profile - J_min) / sigma2_hat
- sigma2_hat = J_min / (n_obs - n_params)
- Identifiable if chi2_profile exceeds 3.84 on both sides of minimum

Profile grid ranges (from RESEARCH.md):
- k0 parameters: log-space, 0.01x to 100x true value (30 points)
- k0_2 specifically: 0.001x to 1000x (wider range due to identifiability risk)
- alpha parameters: linear space, 0.1 to 0.95 (physical bounds, 30 points)

  <action>
    Create `scripts/studies/profile_likelihood_pde.py` following codebase patterns.

    **Structure:**

    1. `ProfileLikelihoodConfig` frozen dataclass:
       - n_points: int = 30
       - params_to_profile: tuple = ("k0_1", "k0_2", "alpha_1", "alpha_2")
       - delta_chi2: float = 3.84 (95% CI, 1 DOF)
       - output_dir: str = "StudyResults/v14/profile_likelihood"
       - noise_percent: float = 2.0
       - noise_seed: int = 0 (use seed 0 for the reference target data)
       - pde_maxiter: int = 25 (L-BFGS-B iterations per re-optimization)
       - verbose: bool = True

    2. `build_profile_grid(param_name: str, true_value: float, n_points: int) -> np.ndarray`:
       - For k0_1: np.logspace(log10(true_val * 0.01), log10(true_val * 100), n_points)
       - For k0_2: np.logspace(log10(true_val * 0.001), log10(true_val * 1000), n_points) -- wider due to identifiability risk
       - For alpha_1, alpha_2: np.linspace(0.1, 0.95, n_points) -- physical bounds

    3. `assess_identifiability(profile_losses: np.ndarray, global_min_loss: float, n_obs: int, n_params: int) -> dict`:
       - Compute sigma2_hat = global_min_loss / (n_obs - n_params)
       - Compute chi2_profile = (profile_losses - global_min_loss) / sigma2_hat
       - Find min_idx = argmin(chi2_profile)
       - left_exceeds = any(chi2_profile[:min_idx] > delta_chi2) if min_idx > 0 else False
       - right_exceeds = any(chi2_profile[min_idx+1:] > delta_chi2) if min_idx < len-1 else False
       - identifiable = left_exceeds and right_exceeds
       - Return dict with identifiable, left_bounded, right_bounded, chi2_profile

    4. `build_fixed_bounds(param_name: str, fixed_val: float, default_bounds: dict) -> dict`:
       - Copy default bounds
       - For the profiled parameter, set lower = upper = fixed_val (pinning it)
       - Return modified bounds dict
       - Parameter mapping: k0_1 -> k0_lower[0]/k0_upper[0], k0_2 -> k0_lower[1]/k0_upper[1], alpha_1 -> alpha_lower[0]/alpha_upper[0], alpha_2 -> alpha_lower[1]/alpha_upper[1]

    5. `run_profile_for_parameter(param_name, config, target_data, global_best) -> dict`:
       - Build grid via build_profile_grid
       - For each grid point: build_fixed_bounds, then call run_bv_multi_observable_flux_curve_inference with joint control_mode and the pinned bounds
       - Initial guess for free parameters: use global_best values
       - Collect profile_losses array
       - Call assess_identifiability
       - Return dict with grid_values, profile_losses, identifiability_result

    6. `generate_profile_plots(param_name, grid_values, chi2_profile, delta_chi2, output_dir)`:
       - Plot chi2_profile vs grid_values
       - Horizontal dashed line at delta_chi2 = 3.84
       - Vertical dashed line at true value
       - For k0 params: use log-scale x-axis
       - Mark identifiability bounds (where chi2 crosses threshold)
       - Save as `profile_{param_name}.png`

    7. `write_profile_csv(param_name, grid_values, profile_losses, chi2_profile, output_dir)`:
       - Columns: grid_value, loss, chi2_profile
       - Save as `profile_{param_name}.csv`

    8. `write_identifiability_summary(results, output_dir)`:
       - `identifiability_summary.csv`: parameter, identifiable, left_bounded, right_bounded, min_loss, ci_lower (grid value where chi2 first drops below threshold from left), ci_upper (grid value where chi2 last drops below threshold from right)
