## Data Storage

**Databases:**
- None. No SQL or NoSQL databases are used.

**File Storage:**
- Local filesystem only
- CSV files for I-V curve data, inference history, and target observations
  - Read/write: `Forward/steady_state/common.py` (`write_phi_applied_flux_csv`, `read_phi_applied_flux_csv`)
  - Read/write: `FluxCurve/bv_run/io.py` (`ensure_bv_target_curve`, `write_bv_history_csv`)
  - Training logs: `Surrogate/nn_training.py` writes epoch-level CSV logs
- Pickle files for RBF surrogate model persistence
  - Serialize: `Surrogate/io.py` (`save_surrogate`, `load_surrogate`)
  - Uses `pickle.HIGHEST_PROTOCOL`
- PyTorch model checkpoints for NN surrogate persistence
  - Save/load via `Surrogate/nn_model.py` `NNSurrogateModel.save()` / `.load()`
  - Stored under `StudyResults/*/nn_ensemble/` directory trees
- PNG files for plots and fit visualizations
  - Generated by `FluxCurve/plot.py`, `Forward/plotter.py`, `writeups/WeekOfMar4/generate_plots.py`
- HDF5 via `h5py` (dependency declared) - Firedrake checkpoint I/O

**Caching:**
- Firedrake TSFC kernel cache (`FIREDRAKE_TSFC_KERNEL_CACHE_DIR`)
- PyOP2 generated code cache (`PYOP2_CACHE_DIR`)
- BV point-solve IC cache: in-memory numpy array cache (`FluxCurve/bv_point_solve/cache.py`) for warm-starting parallel workers from converged solutions

## Computational Backends

**PETSc/SNES (via Firedrake):**
- Nonlinear variational solver for PDE systems
- Configured via `solver_parameters` dicts passed to `fd.NonlinearVariationalSolver`
- Key options: `snes_lag_jacobian`, `snes_lag_jacobian_persists`, Newton line search
- Jacobian assembly and linear solves handled by PETSc's KSP/PC infrastructure

**Firedrake Adjoint (pyadjoint):**
- Automatic differentiation via tape-based adjoint
- `firedrake.adjoint` provides `Control`, `ReducedFunctional`, tape management
- Used for PDE-constrained optimization: gradient of I-V curve mismatch w.r.t. kinetics parameters
- Tape cleared and re-annotated per voltage point in parallel workers (`FluxCurve/bv_parallel.py`)

**Multiprocessing:**
- `concurrent.futures.ProcessPoolExecutor` with `spawn` context for parallel BV point solves
- Each worker imports Firedrake independently, builds its own mesh, and runs forward+adjoint solves
- Pool managed by `BVPointSolvePool` class in `FluxCurve/bv_parallel.py`
