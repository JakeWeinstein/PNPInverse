## Overview

v12 extends the v11 surrogate-warm-started PDE pipeline with neural network
ensemble support and a multi-model comparison framework.  The key question:
**does a 5-member NN ensemble (trained on 3,194 samples) outperform the
RBF-based surrogates for parameter recovery?**

The pipeline retains v11's four-phase architecture:

1. **Phase 1:** Surrogate alpha-only (~0.1 s)
2. **Phase 2:** Surrogate joint 4-param (~0.1 s)
3. **Phase 3:** PDE joint on shallow cathodic (~120 s)
4. **Phase 4:** PDE joint on full cathodic (~120 s)

v12 adds the ability to swap the surrogate used in Phases 1--2 among four
model types, and a `--compare` mode that runs all four in a single invocation
and produces a side-by-side comparison CSV.

### New modules

- **`Surrogate/ensemble.py`** -- `EnsembleMeanWrapper` wraps N `NNSurrogateModel`
  members behind the standard `predict()`/`predict_batch()` API.
  `predict_with_uncertainty()` returns per-point standard deviation across
  ensemble members.  `load_nn_ensemble()` loads member checkpoints from disk.

- **`Surrogate/objectives.py` (extended)** -- `SubsetSurrogateObjective`
  extracted from v11 inline code.  Computes the multi-observable least-squares
  objective on a voltage subset with central FD gradients (8 evaluations per
  gradient for 4 parameters).

## Analysis

### NN ensemble strengths

1. **Lowest surrogate loss** (5.87e-5 vs 6.26e-5 for RBF) -- the ensemble
   mean provides a smoother, more accurate I-V curve fit than any single
   interpolant.

2. **Best transfer coefficient recovery** -- alpha_1 = 3.54% and alpha_2 =
   3.48%, approximately 2x better than the RBF models (5--8%).  The ensemble
   averaging suppresses high-frequency interpolation noise that biases alpha
   recovery in the RBF models.

3. **Best k0_1 recovery** (6.88%) -- beats all competitors (RBF: 10.03%,
   POD-RBF-log: 11.77%, POD-RBF-nolog: 12.16%).

### NN ensemble weakness

4. **Worst k0_2 recovery** (11.85%) -- substantially worse than POD-RBF-log
   (1.64%), POD-RBF-nolog (3.08%), and RBF (9.41%).  The peroxide current
   I_pxd = -(R1 - R2) * I_scale involves cancellation of two similar
   quantities, and the POD-RBF-log model's explicit log1p transform better
   captures this structure.

### Model trade-off landscape

The familiar k0_1--k0_2 anti-correlation persists across model types:

- **NN ensemble:** best k0_1 (6.88%) but worst k0_2 (11.85%)
- **POD-RBF-log:** best k0_2 (1.64%) but worst k0_1 (11.77%)
- **RBF baseline:** most balanced (10.03%, 9.41%) but highest max error among
  balanced models

No surrogate achieves < 10% on all four parameters simultaneously from
Phase 2 alone -- this remains the job of PDE refinement in Phases 3--4.

### Ensemble uncertainty

At the Phase 2 optimum, the NN ensemble reports:
- Mean current density std: 1.96e-4
- Mean peroxide current std: 9.78e-4

The 5x higher uncertainty on peroxide current is consistent with the known
cancellation-error difficulty: the ensemble members disagree most on the
I_pxd = -(R1 - R2) observable.

### Minimax comparison (lowest worst-case error)

| Model | Max single-param error | Which param |
|-------|----------------------|-------------|
| RBF baseline | 10.03% | k0_1 |
| NN ensemble | 11.85% | k0_2 |
| POD-RBF-nolog | 12.16% | k0_1 |
| POD-RBF-log | 11.77% | k0_1 |

The RBF baseline has the lowest minimax error (10.03%), despite its higher
total loss.  However, the NN ensemble has the lowest *mean* error across all
4 parameters: (6.88 + 11.85 + 3.54 + 3.48) / 4 = 6.44%, versus RBF's
(10.03 + 9.41 + 5.44 + 5.93) / 4 = 7.70%.


## PDE Refinement (Phases 3--4): Complete

The full PDE pipeline (Phases 3--4) used the NN ensemble P2 result as warm
start.  Both phases ran L-BFGS-B optimization against the Firedrake PDE
solver.

### Phase-by-phase summary

| Phase | k0_1 err | k0_2 err | alpha_1 err | alpha_2 err | Loss | Time |
|-------|----------|----------|-------------|-------------|------|------|
| P1 alpha (nn-ensemble) | 295.83% | 850.00% | 49.65% | 78.50% | 5.608e-04 | 0.1s |
| P2 joint (nn-ensemble) | 6.88% | 11.85% | 3.54% | 3.48% | 5.866e-05 | 0.6s |
| **P3 PDE shallow** | **6.80%** | **11.80%** | **3.88%** | **3.21%** | **5.374e-05** | 100.7s |
| P4 PDE full cathodic | 16.64% | 5.97% | 8.14% | 11.68% | 1.251e-04 | 536.9s |

### Best result: Phase 3 (PDE shallow)

Phase 3 was selected as the best phase by the minimax criterion (lowest
worst-case error across all 4 parameters: 11.80%).

| Parameter | True | Estimated | Rel. Error |
|-----------|------|-----------|------------|
| k0_1 | 1.2632e-3 | 1.3490e-3 | 6.80% |
| k0_2 | 5.2632e-5 | 4.6420e-5 | 11.80% |
| alpha_1 | 0.6267 | 0.6027 | 3.88% |
| alpha_2 | 0.5000 | 0.4839 | 3.21% |

Redimensionalized values (K_scale = 1.900e-05 m/s):
- k0_1: true = 2.4000e-08, est = 2.5632e-08 m/s (err 6.80%)
- k0_2: true = 1.0000e-09, est = 8.8198e-10 m/s (err 11.80%)
- alpha_1: true = 0.627, est = 0.603 (err 3.88%)
- alpha_2: true = 0.500, est = 0.484 (err 3.21%)

### Phase 4 details

Phase 4 expanded the voltage range to the full cathodic window and ran 25
L-BFGS-B evaluations warm-started from the P3 optimum.  It converged with
gradient norm 3.08e-06 and achieved a lower loss on k0_2 (5.97%) but at the
cost of degraded k0_1 (16.64%) and alpha errors (8--12%).

This is consistent with v7/v9 behavior: the full cathodic range introduces
additional local minima in the PDE objective landscape, particularly from the
diffusion-limited plateau where the I-V curve is insensitive to kinetic
parameters.

### P3 vs P4 verdict

Phase 3 is the clear winner.  Its max single-parameter error (11.80%) is
substantially lower than Phase 4's (16.64%).  The wider voltage range in P4
did improve k0_2 from 11.80% to 5.97%, but this came at the expense of k0_1
and both transfer coefficients -- the hallmark of overfitting to the
diffusion-limited regime.

v12 PDE achieves the best k0_1 (6.80%) and alpha (3.2--3.9%) recovery across
all versions, but trails v7 and v11 on k0_2.  The max error (11.8%) is higher
than v11's 9.4%, entirely due to the stubborn k0_2 difficulty.  This suggests
the NN ensemble warm start lands in a different basin of the PDE objective
landscape -- one that favors k0_1/alpha accuracy at the expense of k0_2.

Key observation: the 7x data expansion (445 -> 3194 samples) did **not**
improve the RBF baseline's P2 accuracy meaningfully (v9: 8.76% max vs v12
RBF: 10.03% max).  The training data was already sufficient for RBF
interpolation; the bottleneck is the interpolation method itself.  The NN
ensemble benefits from the larger dataset through better generalization of the
peroxide current structure, but its gains manifest primarily in alpha and k0_1
recovery, not k0_2.


## Implementation Architecture

```
v12 Script
  |
  +-- _load_model(model_type, args)
  |     |-- "nn-ensemble" -> load_nn_ensemble(nn_dir/design) -> EnsembleMeanWrapper
  |     |-- "rbf"         -> pickle.load(model_rbf_baseline.pkl) -> BVSurrogateModel
  |     |-- "pod-rbf-log" -> pickle.load(model_pod_rbf_log.pkl) -> PODRBFSurrogateModel
  |     +-- "pod-rbf-nolog" -> pickle.load(model_pod_rbf_nolog.pkl) -> PODRBFSurrogateModel
  |
  +-- Phase 1: AlphaOnlySurrogateObjective  (fix k0, optimize alpha via L-BFGS-B)
  +-- Phase 2: SubsetSurrogateObjective     (joint 4-param on shallow cathodic subset)
  +-- [--compare]: repeat P1-P2 for all 4 model types, emit model_comparison_v12.csv
  +-- Phase 3: PDE joint on shallow cathodic (warm from P2)
  +-- Phase 4: PDE joint on full cathodic    (warm from P3)
  +-- Best-of selection: min(P3_max_err, P4_max_err)
```

### EnsembleMeanWrapper API

```python
class EnsembleMeanWrapper:
    """Wraps N NNSurrogateModel members; prediction = mean across members."""

    def predict(k0_1, k0_2, alpha_1, alpha_2) -> dict  # single point
    def predict_batch(parameters: ndarray) -> dict      # (M, 4) batch
    def predict_with_uncertainty(...) -> dict            # + cd_std, pc_std

    @property phi_applied -> ndarray   # voltage grid
    @property n_eta -> int             # number of voltage points
    @property is_fitted -> bool        # always True
    training_bounds: dict              # merged min(lo)/max(hi) across members
```

2. **Hybrid surrogate strategy** -- The POD-RBF-log model's 1.64% k0_2 error
   suggests it captures the peroxide current structure better than either the
   NN ensemble or plain RBF.  A hybrid that uses the NN ensemble for current
   density and POD-RBF-log for peroxide current could combine the best of
   both.

3. **Ensemble size study** -- The current ensemble uses 5 members.  Evaluate
   whether 3 or 10 members changes the accuracy-uncertainty tradeoff.

4. **Noise robustness** -- Run the v12 comparison at 0% noise to separate
   surrogate fidelity from noise-induced error, as was done for v11 (where
   noise-free achieved 1.07% max error).

5. **Phase 4 regularization** -- The full cathodic range consistently degrades
   results across v7, v9, and v12.  Consider weighting the objective to
   de-emphasize the diffusion-limited plateau, or restricting Phase 4 to a
   slightly narrower range that avoids the worst local minima.
