### Re point 7 — Newton-from-U_manuf may degenerate to interpolation test

**Accept and material.** This is the biggest substantive risk surfaced
by GPT in R1.

**Why it matters in detail:** at u_exact (continuum), all sources cancel
exactly, so F_res(u_exact) ≡ 0. The FE problem solves for U_h ≈
Π_FE(u_exact) (the projection). F_res(U_manuf_h) ≠ 0 in general
(interpolation error projects into residual at O(h²) for CG1 in L2),
so Newton has work to do. **But:** if the residual at U_manuf_h is
already below `snes_atol`, SNES exits in 0 iterations and the "L2
error" is just the interpolation error (also O(h²)), giving the *right
convergence rate* even if the source builder is wildly broken.

**Mitigation (added to step 10 pilot):**

10.5 **Perturbed-IC pilot**: at N=32, solve from `U_init = U_manuf + ε·𝓝`
where ε = 1e−3 and 𝓝 is a fixed random Function (seeded). Newton must:
- Take ≥ 3 iterations (so the residual reduction is genuinely tested).
- Converge to L2 error within 1% of the unperturbed-IC L2 error.

10.6 **Iteration-count sanity** in `run_mms`: assert `newton_iterations
≥ 2` for the unperturbed runs at every mesh size. Document the observed
iteration counts in the test class docstring; if any mesh hits 0 or 1
iteration, investigate before merging.

10.7 **Residual-reduction check** at every mesh: record
`F_res_l2_initial / F_res_l2_final`. Reduction must be > 100×; otherwise
the test is operating below the discretization-error signal floor.

WHAT TO DO: Use deterministic smooth analytic perturbations that vanish on essential-BC boundaries, e.g. `eps*sin(pi*x)*sin(pi*y)` variants per component. Apply BCs to the initial guess before solve.

WHY: If the initial residual is already small, the ratio is noise-dominated. This recreates the same false-failure risk as the iteration-count floor.
WHAT TO DO: Apply residual-reduction criteria to the perturbed-start pilot only, or guard with an absolute initial-residual floor.

WHAT TO DO: Compare both closures using the same inputs. Either build the independent closure from `ctx["ci_exprs"]`/`phi` for this diagnostic only, or compare analytic-vs-production with an interpolation-error-scaled tolerance, not `1e-9`.
